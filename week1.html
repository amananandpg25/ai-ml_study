4. Type Conversion (Casting)

You can convert between types using int(), float(), str(), bool(). Critical rule: input() always returns a string — you must convert it before doing math.

💡 Why does Python use references? When you write b = a for a list, Python doesn't copy the data — it creates a second name pointing to the same memory location. This is efficient (no wasted RAM for large datasets), but it means modifying b also modifies a. Think of it like two people sharing the same Google Doc: edits by one are visible to the other instantly.
python — type_casting.py
marks_str = "95"               # This is TEXT, not a number
marks_int = int(marks_str)     # Now it's the number 95
marks_flt = float(marks_str)   # Now it's 95.0

print(marks_int + 5)           # 100  ✅ math works
print(marks_str + "5")         # "955" ⚠️ string concatenation!

# input() always returns str — ALWAYS convert before math
user_age = int(input("Enter age: "))  # convert immediately
birth_year = 2026 - user_age          # now this works ✅

# bool() truthy/falsy rules (important for ML conditionals)
print(bool(0))      # False — zero is always falsy
print(bool(""))     # False — empty string is falsy
print(bool(42))     # True — any non-zero number is truthy
print(bool("hi"))   # True — any non-empty string is truthy

5. f-strings — The Professional Way to Print

f-strings (formatted string literals) provide a clean, readable, and fast way to embed expressions inside string literals using minimal syntax.

python — fstrings.py
name = "Arjun"
city = "Mumbai"
score = 94.567

# Old way — ugly and error-prone
print("Hello " + name + " from " + city)

# f-string — clean, readable, professional
print(f"Hello {name} from {city}!")

# Format numbers with f-strings
print(f"Score: {score:.2f}")        # 94.57 — 2 decimal places
print(f"Score: {score:.0f}")        # 95 — rounded to int
print(f"Score: {score:10.2f}")      # right-aligned in 10 chars

# You can put expressions inside {}
print(f"Next year age: {21 + 1}")   # 22
print(f"Upper: {name.upper()}")     # ARJUN
⚠️ Top 3 Day-1 Mistakes: 1. = vs == → age = 21 assigns. age == 21 compares. Writing if age = 21 crashes with SyntaxError: invalid syntax.
2. Forgetting to convert input() → x = input("num: ") then x + 1 crashes with TypeError: can only concatenate str (not "int") to str. Fix: x = int(input("num: "))
3. true vs True → Python is case-sensitive. true, false, null are NOT valid Python. It must be True, False, None.

where P = principal, r = monthly rate (annual_rate/12/100), n = months
Hint: ** operator power ke liye use hota hai Python mein. (1+r)**n matlab (1+r) ki power n. monthly_rate = annual_rate / 12 / 100.

# emi_calculator.py — Smart EMI Calculator
# Skills: input(), float(), int(), arithmetic operators, ** (power), round(), f-strings

print("   🏦 SMART EMI CALCULATOR")
print("═" * 40)

principal = float(input("Loan Amount (₹): "))

# Convert annual rate to monthly rate
monthly_rate = annual_rate / 12 / 100

# EMI formula: P * r * (1+r)^n / ((1+r)^n - 1)

emi = numerator / denominator

1. Lists — Ordered, Mutable Collections

Lists are the most versatile built-in sequence type in Python, allowing you to store an ordered collection of items that can be dynamically modified.

⚠️ Gotcha: Shallow vs Deep Copy Assignment (b = a) creates a reference, not a copy. b = a.copy() creates a shallow copy (nested objects still share references). Use import copy; b = copy.deepcopy(a) for fully independent nested copies. This is a top Python interview pitfall.
python — lists.py
# Create lists with square brackets
ipl_teams = ["MI", "CSK", "RCB", "KKR"]
scores = [89, 76, 94, 61]

# Indexing: 0 = first, -1 = last
print(ipl_teams[0])      # "MI"
print(ipl_teams[-1])     # "KKR"
print(ipl_teams[1:3])    # ["CSK", "RCB"] — stop index NOT included!

# Modifying lists
ipl_teams.append("SRH")        # add to end
ipl_teams.insert(1, "PBKS")    # insert at index 1
ipl_teams.remove("RCB")        # remove first occurrence of value
last = ipl_teams.pop()          # remove & return last item
ipl_teams.sort()                # sort alphabetically IN-PLACE (changes original)
copy_sorted = sorted(ipl_teams) # sort and return NEW list (original unchanged)

# Useful operations
print(len(ipl_teams))    # count of items
print("MI" in ipl_teams) # True — membership check
print(scores.index(94)) # 2 — position of value 94

# List comprehension — most important Python pattern for ML
passed = [s for s in scores if s >= 75]   # [89, 76, 94] filter
doubled = [s * 2 for s in scores]         # transform every item
In ML, your entire training dataset is a list of lists (or NumPy array, which IS a list underneath). list comprehensions are used to filter data, normalize values, and transform labels — you'll write dozens of these in Pandas and scikit-learn.

3. Sets — Unique Values, Fast Membership

Sets are unordered collections of unique elements, highly optimized for $O(1)$ fast membership testing and mathematical operations.

python — sets.py
# Sets store only UNIQUE values — auto-removes duplicates
students_a = {"Raj", "Priya", "Amit", "Raj"}
print(students_a)   # {"Raj", "Priya", "Amit"} — no duplicate Raj

# Remove duplicates from a list (classic trick)
marks = [85, 92, 85, 78, 92]
unique = list(set(marks))   # [85, 92, 78]

# Set operations — very useful in data analysis
python_devs = {"Raj", "Priya", "Amit"}
ml_devs = {"Priya", "Sara", "Amit"}

print(python_devs | ml_devs)   # union — all unique names
print(python_devs & ml_devs)   # intersection — know BOTH Python + ML
print(python_devs - ml_devs)   # difference — Python only, not ML

# Membership check is O(1) — instant even for 1 million items
print("Priya" in python_devs)  # True

4. Dictionaries — The Most Important Structure in ML

⚡ Big-O Time Complexity & ML Vocabulary Mapping:
• Lookup: $O(1)$ average time complexity. Dictionaries are implemented as hash tables. Looking up a key takes a constant amount of time, regardless of whether the dictionary has 10 keys or 10 million keys.
• List lookup vs. Dict lookup: Checking if an element exists in a list takes $O(n)$ time because Python has to scan the entire list sequentially. In ML, always use a dictionary or a set for lookup tables (like a vocabulary dictionary mapping words to IDs in NLP: vocab = {'hello': 0, 'world': 1}).
• Hash Collisions: Under the hood, if two different keys hash to the same index, Python resolves this using open addressing, which can degrade lookup performance to $O(n)$ in the absolute worst-case scenario (extremely rare).
python — dicts.py
# Dicts store key:value pairs — like a real dictionary
student = {
    "name": "Arjun Mehta",
    "age": 21,
    "gpa": 8.9,
    "subjects": ["Python", "ML", "Stats"]  # list as a value!
}

# Access values by key
print(student["name"])              # "Arjun Mehta"
print(student.get("city", "N/A"))   # "N/A" — safe, no crash if key missing

# Add / update / delete
student["city"] = "Bangalore"         # add new key
student["gpa"] = 9.1                 # update existing key
del student["age"]                   # remove key

# Iterate over dict
for key, value in student.items():
    print(f"{key}: {value}")

# Dict comprehension — builds a dict from a sequence
grades = {"Raj": 88, "Priya": 92, "Amit": 75}
passed = {name: g for name, g in grades.items() if g >= 80}

# ml_utils.py — Reusable ML utility functions

def normalize(values):
    """Scale list to 0-1 range (min-max normalization)."""
    lo, hi = min(values), max(values)
    return [round((v - lo) / (hi - lo), 4) for v in values]

def clamp(value, lo=0, hi=1):
    """Clip value to [lo, hi] range."""
    return max(lo, min(hi, value))

def mean(*args):
    """Compute mean of any number of values."""
    return sum(args) / len(args)

def accuracy(predictions, labels):
    """Compute classification accuracy."""
    correct = sum(1 for p, l in zip(predictions, labels) if p == l)
    return correct / len(labels)

def one_hot(label, num_classes):
    """Convert class index to one-hot vector."""
    vec = [0] * num_classes
    vec[label] = 1
    return vec

# Test all functions

1. Reading and Writing Files

⚡ Memory-Efficient Data Loading via Generators ():
When training ML models on datasets that are larger than your computer's RAM, loading the entire file into a Python list using f.readlines() will crash your kernel (Out Of Memory error).
Instead, use a **generator** with the yield keyword. A generator yields one row at a time, keeping only a single line in memory: 
def stream_dataset(file_path):
    with open(file_path, 'r') as f:
        for line in f:

# mini_pipeline.py — sklearn-style OOP

class MinMaxScaler:
    """Scales features to [0, 1] range. Just like sklearn.preprocessing.MinMaxScaler."""
    def __init__(self):
        self.min_ = None
        self.max_ = None

    def fit(self, data):    # learn the min and max from data
        self.min_ = min(data)
        self.max_ = max(data)
        return self              # return self for chaining

    def transform(self, data):   # apply learned scaling
        if self.min_ is None:
            raise ValueError("Call fit() before transform()!")
        r = self.max_ - self.min_
        return [round((x - self.min_) / r, 4) for x in data]

    def fit_transform(self, data):   # fit and transform in one call
        return self.fit(data).transform(data)


class ThresholdClassifier:
    """Classifies values above threshold as 1, else 0."""
    def __init__(self, threshold=0.5):
        self.threshold = threshold

    def predict(self, data):
        return [1 if x >= self.threshold else 0 for x in data]

    def score(self, data, labels):
        preds = self.predict(data)
        correct = sum(1 for p, l in zip(preds, labels) if p == l)
        return correct / len(labels)


# Use the mini pipeline

labels =     [0,  1,  1,  0,  1,  0,  1]

scaled = scaler.fit_transform(raw_scores)
print("Scaled:", scaled)

clf = ThresholdClassifier(threshold=0.5)
preds = clf.predict(scaled)
print("Predictions:", preds)
print(f"Accuracy: {clf.score(scaled, labels):.0%}")

2. Vectorized Operations — Why NumPy is 100x Faster

Vectorization bypasses slow Python loop operations by executing math calculations in parallel compiled C code, drastically accelerating numerical computations.

python
scores = np.array([85, 92, 78, 95, 88])
