Imagine you are in a library searching for 'Indian spices'. The query (Q) is your search term. The database cards (K) are the index descriptions of all the books in the library. The value (V) is the actual content pages of the books. Self-attention does a dot-product search of Q against all Ks, finds the best matches, and pulls a weighted combination of their Vs.

🧠 Theory

1. Information Bottleneck in RNNs

In standard sequence-to-sequence models (e.g., LSTMs), an entire input sequence of length T is encoded into a fixed-dimensional context vector $h_T$ ∈ ℝ^d. For large T (e.g., 500 words), compressing a distribution of variable length into a static bounded space leads to high information loss. The Jacobian matrix of hidden states across time steps reveals exponentially decaying singular values, proving that long-term dependencies vanish.

Think of RNN like this: Ek 100-page ki book padh kar poori book ko ek single paragraph summary mein memorize karna, aur phir kisi bhi question ka answer dena. Wahi dusri taraf Attention matlab open-book exam—jahan aap bina memory pressure ke directly page ke us sentence par focus kar sakte ho jo question ke liye sabse zyada relevant hai.

2. $Q$, $K$, $V$ Projections

Attention resolves this by allowing the decoder to access a soft-addressed memory of all encoder states. We project the input embeddings $X$ ∈ ℝ^(T × $d_{\text{model}}$) into three distinct sub-spaces using learned weight matrices:

$Q$ = $X$ $W_Q$ (where $W_Q$ ∈ ℝ^($d_{\text{model}}$ × $d_k$)) $K$ = $X$ $W_K$ (where $W_K$ ∈ ℝ^($d_{\text{model}}$ × $d_k$)) $V$ = $X$ $W_V$ (where $W_V$ ∈ ℝ^($d_{\text{model}}$ × $d_v$)) $Q$ represents what we are searching for, $K$ acts as the database index, and $V$ is the actual feature content.

3. Deriving Scaled Dot-Product Attention

The unscaled attention scores are given by S = $Q$ $K$^T. If components of $Q$ and $K$ are independent random variables with mean 0 and variance 1, the dot product will have mean 0 and variance $d_k$. For a large $d_k$ (e.g., 64), the variance is 64.

Passing a vector with high variance into the Softmax function pushes the values to the tails of the exponential curve, where the derivative of softmax approaches zero. This causes gradient saturation and halts learning. To solve this, we scale by the standard deviation √$d_k$.

$$Attention($Q$, $K$, $V$) = Softmax( ($Q$ $K$^T) / √$d_k$ ) $V$$$

python — causal_attention.py
import torch
import torch.nn.functional as F
import math

def causal_self_attention(q, k, v, mask=True):
    # q, k, v shapes: [batch_size, seq_len, d_k]
    d_k = q.size(-1)
    seq_len = q.size(-2)

    # 1. Dot product: (Q @ K.T) / sqrt(d_k)
    scores = torch.matmul(q, k.transpose(-2, -1)) / math.sqrt(d_k)

    # 2. Causal Masking (autoregressive property)
    if mask:
        # lower triangular matrix of ones
        causal_mask = torch.tril(torch.ones(seq_len, seq_len, device=q.device))
        # replace 0s with -infinity
        scores = scores.masked_fill(causal_mask == 0, float('-inf'))

    # 3. Softmax along the last dimension (keys)
    attn_weights = F.softmax(scores, dim=-1)

    # 4. Multiply with values
    output = torch.matmul(attn_weights, v)
    return output, attn_weights

python — stable_softmax.py
import numpy as np

def softmax_naive(x):
    exps = np.exp(x)
    return exps / np.sum(exps)

def softmax_stable(x):
    # Subtract the max value to prevent overflow (e^300 = inf)
    shifted_x = x - np.max(x)
    exps = np.exp(shifted_x)
    return exps / np.sum(exps)

x1 = np.array([10.0, 20.0, 30.0])
print("Naive:", softmax_naive(x1))
print("Stable:", softmax_stable(x1))

x2 = np.array([100.0, 200.0, 300.0])
# print(softmax_naive(x2)) # Prints [nan, nan, nan] because of inf / inf
print("Stable large:", softmax_stable(x2))
💡 Mathematical Proof: $\\text{Softmax}(x)_i = \\frac{e^{x_i - \\max(x)}}{\\sum_j e^{x_j - \\max(x)}}$. By subtracting $\\max(x)$, the largest element becomes $0$ ($e^0 = 1$), and all other elements are negative (exponents $< 1$), completely avoiding overflow errors.

python — flash_attention.py
import torch
import math

def flash_attention_naive(Q, K, V, B_q=2, B_k=2):
    T, d = Q.shape
    O = torch.zeros(T, d)
    L = torch.zeros(T, 1)
    M = torch.fill(torch.zeros(T, 1), float('-inf'))

    for i in range(0, T, B_q):
        q_block = Q[i:i+B_q]

        for j in range(0, T, B_k):
            k_block = K[j:j+B_k]
            v_block = V[j:j+B_k]

            S = torch.matmul(q_block, k_block.t()) / math.sqrt(d)
            m_block = torch.max(S, dim=-1, keepdim=True)[0]

            new_M = torch.max(M[i:i+B_q], m_block)
            exp_old = torch.exp(M[i:i+B_q] - new_M)
            exp_new = torch.exp(S - new_M)

            new_L = exp_old * L[i:i+B_q] + torch.sum(exp_new, dim=-1, keepdim=True)
            O[i:i+B_q] = (O[i:i+B_q] * exp_old * L[i:i+B_q] + torch.matmul(exp_new, v_block)) / new_L

            L[i:i+B_q] = new_L
            M[i:i+B_q] = new_M

    return O

Q = torch.randn(6, 8)
K = torch.randn(6, 8)
V = torch.randn(6, 8)

standard_attn = torch.softmax(torch.matmul(Q, K.t()) / math.sqrt(8), dim=-1) @ V
tiled_attn = flash_attention_naive(Q, K, V)

print("Max Difference:", torch.max(torch.abs(standard_attn - tiled_attn)).item())
💡 FlashAttention Logic: In standard attention, the entire $N \times N$ matrix of softmax weights is kept in GPU SRAM/DRAM, which is memory-bound. FlashAttention keeps running statistics of maximums $M$ and sums $L$ to compute it tile-by-tile, making the memory footprint $O(N)$ instead of $O(N^2)$.

2. Multi-Head Attention

Instead of calculating a single attention distribution, Multi-Head Attention splits the embeddings into h heads, computes attention independently in parallel, and concatenates the results. This allows the model to simultaneously attend to different representation subspaces (e.g., one head for syntax, one head for subject-verb relationships).

3. Rotary Positional Embeddings (RoPE)

The original Transformer used Absolute Positional Encodings (adding sine/cosine waves to token vectors). Modern LLMs (like LLaMA and Mistral) use RoPE, which encodes relative position by rotating the query and key representations in a 2D plane by an angle proportional to their absolute position.

$$RoPE(x_m) = x_m * cos(mθ) + [-x_m(2), x_m(1)] * sin(mθ)$$ Where m is the absolute position. The inner product between two rotated vectors naturally decays relative to their distance (m - n), giving strong relative position bias.

4. Pre-LN vs Post-LN

Original Transformers used Post-LayerNorm (LayerNorm after residual connections). This led to severe gradient vanishing during early training, requiring a long learning rate warmup. Modern architectures (GPT-2, LLaMA) use Pre-LayerNorm (LayerNorm inside the residual block), allowing for stable training without warmup.

5. Autoregressive Generation & KV Caching

During inference, decoder models generate tokens one by one. Recomputing attention for all past tokens at every step is computationally expensive $O(N^2)$. KV Caching solves this by storing the previously computed Key and Value vectors for past tokens in memory. At step t, we only compute the Query, Key, and Value for the new token, and append its $K$ and $V$ to the cache, reducing time complexity per step to $O(N)$.

        self.head_dim = d_model // n_heads

        self.q_proj = nn.Linear(d_model, d_model, bias=False)
        self.k_proj = nn.Linear(d_model, d_model, bias=False)
        self.v_proj = nn.Linear(d_model, d_model, bias=False)
        self.out_proj = nn.Linear(d_model, d_model, bias=False)

    def forward(self, x, past_key_values=None):
        B, T, C = x.shape
        q = self.q_proj(x).view(B, T, self.n_heads, self.head_dim).transpose(1, 2)
        k = self.k_proj(x).view(B, T, self.n_heads, self.head_dim).transpose(1, 2)
        v = self.v_proj(x).view(B, T, self.n_heads, self.head_dim).transpose(1, 2)

        if past_key_values is not None:
            past_k, past_v = past_key_values
            k = torch.cat([past_k, k], dim=-2)
            v = torch.cat([past_v, v], dim=-2)

        new_past_key_values = (k, v)
        total_T = k.size(-2)

        scores = torch.matmul(q, k.transpose(-2, -1)) / math.sqrt(self.head_dim)
        causal_mask = torch.tril(torch.ones(T, total_T, device=x.device))
        scores = scores.masked_fill(causal_mask == 0, float('-inf'))

        attn = torch.softmax(scores, dim=-1)
        out = torch.matmul(attn, v).transpose(1, 2).contiguous().view(B, T, C)
        return self.out_proj(out), new_past_key_values

def precompute_freqs_cis(dim, end, theta=10000.0):
    freqs = 1.0 / (theta ** (torch.arange(0, dim, 2)[: (dim // 2)].float() / dim))
    t = torch.arange(end, device=freqs.device)
    freqs = torch.outer(t, freqs).float()
    freqs_cis = torch.polar(torch.ones_like(freqs), freqs)
    return freqs_cis

def apply_rotary_emb(xq, xk, freqs_cis):
    xq_ = torch.view_as_complex(xq.float().reshape(*xq.shape[:-1], -1, 2))
    xk_ = torch.view_as_complex(xk.float().reshape(*xk.shape[:-1], -1, 2))
    freqs_cis = freqs_cis.view(1, xq_.size(1), 1, xq_.size(3))

    xk_out = torch.view_as_real(xk_ * freqs_cis).flatten(3)

🧠 Theory & Engineering Deep Dive

1. Why Bidirectionality Is the Key Innovation

GPT (OpenAI, 2018) used a Decoder-only Transformer — trained left-to-right, predicting the next token. This creates representations where each token can only "see" the tokens before it. For a word like "bank" in "I went to the bank to deposit money", a left-to-right model has only seen "I went to the" — not enough context to disambiguate financial vs. river bank.

BERT uses only the Encoder stack, which applies fully bidirectional self-attention — each token attends to every other token in both directions simultaneously. The representation of "bank" is computed using the full context including "deposit money" which follows it. This produces dramatically richer contextual representations, especially for tasks requiring deep understanding like QA, NLI, and coreference resolution.

$$h_i^{(l)} = \\text{TransformerEncoder}\\!\\left(h_1^{(l-1)}, \\ldots, h_n^{(l-1)}\\right)_i$$ Each token representation at layer l is a function of ALL tokens at layer l-1. Contrast with GPT where h_i^(l) depends only on h_1...h_i^(l-1).

2. Masked Language Modeling — The 80/10/10 Strategy

BERT cannot use standard next-token prediction (that would require a causal mask, destroying bidirectionality). Instead, it uses Masked Language Modeling (MLM): randomly select 15% of tokens per sequence, then replace them as follows — 80% with [MASK], 10% with a random token, 10% left unchanged. The model must predict the original token for all 15% selected positions.

Selection (15%)What happensProportionRationale
SelectedReplaced with [MASK]80%Primary learning signal — predict masked tokens
SelectedReplaced with random token10%Forces model to maintain contextual representation for all tokens (not just [MASK])
SelectedLeft unchanged10%Prevents model from only focusing on [MASK] positions
Not selected (85%)Unchanged85%Context for predicting the masked tokens

The 80/10/10 split is critical. If we only ever replaced with [MASK], the model would learn "only attend to [MASK] positions" — but at fine-tuning time, there are no [MASK] tokens. The 10% random + 10% unchanged forces the model to maintain useful representations for every token in every context.

3. Special Tokens and Input Construction

Every BERT input has a specific structure. Understanding exactly how inputs are constructed is essential for fine-tuning:

  • [CLS] (Classification Token): Always the first token. After 12/24 layers of self-attention, the [CLS] representation has attended to all tokens and accumulates a sequence-level summary. For classification tasks, only this representation is passed to the output layer.
  • [SEP] (Separator Token): Marks the end of a sentence or the boundary between two sentences in a pair. Required even for single-sentence inputs (appended at end).
  • Token Type IDs (Segment Embeddings): 0 for all tokens in sentence A, 1 for all tokens in sentence B. Allows BERT to distinguish which sentence each token belongs to in pair tasks.
  • Attention Mask: 1 for real tokens, 0 for [PAD] tokens (used when batching sequences of different lengths). Prevents the model from attending to padding positions.

import torch

class MLMDataCollator:
    def __init__(self, tokenizer, mlm_probability=0.15):
        self.tokenizer = tokenizer
        self.mlm_probability = mlm_probability

    def __call__(self, examples):
        input_ids = torch.stack([ex["input_ids"] for ex in examples])
        labels = input_ids.clone()

        probability_matrix = torch.full(labels.shape, self.mlm_probability)
        special_tokens_mask = [
            self.tokenizer.get_special_tokens_mask(val, already_has_special_tokens=True)
            for val in labels.tolist()
        ]
        special_tokens_mask = torch.tensor(special_tokens_mask, dtype=torch.bool)
        probability_matrix.masked_fill_(special_tokens_mask, value=0.0)

        masked_indices = torch.bernoulli(probability_matrix).bool()
        labels[~masked_indices] = -100

        indices_replaced = torch.bernoulli(torch.full(labels.shape, 0.8)).bool() & masked_indices
        input_ids[indices_replaced] = self.tokenizer.convert_tokens_to_ids("[MASK]")

        indices_random = torch.bernoulli(torch.full(labels.shape, 0.5)).bool() & masked_indices & ~indices_replaced
        random_words = torch.randint(len(self.tokenizer), labels.shape, dtype=torch.long)
        input_ids[indices_random] = random_words[indices_random]

        return {
            "input_ids": input_ids,
            "attention_mask": torch.stack([ex["attention_mask"] for ex in examples]),

from collections import Counter
import math

def get_ngrams(words, n):
    return [tuple(words[i:i+n]) for i in range(len(words)-n+1)]

def bleu_precision(candidate, reference, n):
    cand_ngrams = get_ngrams(candidate, n)
    ref_ngrams = get_ngrams(reference, n)
    if not cand_ngrams:
        return 0.0
    cand_counts = Counter(cand_ngrams)
    ref_counts = Counter(ref_ngrams)

    clipped_matches = 0
    for ngram, count in cand_counts.items():
        clipped_matches += min(count, ref_counts[ngram])
    return clipped_matches / len(cand_ngrams)

def compute_bleu(candidate_text, reference_text):
    cand_words = candidate_text.lower().split()
    ref_words = reference_text.lower().split()
    p1 = bleu_precision(cand_words, ref_words, 1)
    p2 = bleu_precision(cand_words, ref_words, 2)
    if p1 == 0 or p2 == 0:
        return 0.0
    c = len(cand_words)
    r = len(ref_words)
    bp = 1.0 if c > r else math.exp(1 - r / c)
    score = bp * math.exp(0.5 * math.log(p1) + 0.5 * math.log(p2))
    return score

🧠 Theory

1. Subword Tokenization Algorithms

Out of Vocabulary (OOV) issues break word-level tokenization. Modern models use subwords:

  • BPE (Byte Pair Encoding): Merges the most frequent pairs of bytes/characters iteratively. Used in GPT.
  • WordPiece: Merges pairs that maximize the language model likelihood (Score = freq(ab) / (freq(a)*freq(b))). Used in BERT.
  • SentencePiece (Unigram): Starts with a massive vocabulary and iteratively prunes tokens that cause the smallest drop in total probability. Used in LLaMA / ALBERT.

2. Fast Tokenizers in Rust

Hugging Face's `tokenizers` library is written in Rust, utilizing a highly parallelized pipeline allowing encoding of 1GB of text in under 20 seconds. The pipeline consists of:

  • Normalizer: Lowercases, strips accents, applies unicode normalization (NFC, NFKC).
  • Pre-tokenizer: Splits text into initial chunks (usually by whitespace and punctuation) to restrict subword boundaries.
  • Model: Applies the actual algorithm (BPE, WordPiece) to the chunks.
  • Post-processor: Injects special tokens like [CLS] and [SEP] for BERT.

Decoder-only models (like GPT-2) are like playing word association games where you are only allowed to look at the words already spoken. You must write the next word using a causal mask that hides all future cards in the deck, forcing the model to generate one step at a time.

🧠 Theory

1. The Need for PEFT

Full fine-tuning requires updating and storing optimizer states for every parameter. For a 7B model, Adam optimizer states take 14GB of VRAM on top of the model weights. Parameter-Efficient Fine-Tuning (PEFT) methods reduce trainable parameters by >99% while maintaining 90-100% of performance.

2. LoRA (Low-Rank Adaptation)

LoRA hypothesizes that the change in weights (ΔW) during fine-tuning has a low "intrinsic rank". Instead of updating the massive weight matrix $W$ ∈ ℝ^(d × k) directly, we freeze $W$ and inject trainable rank decomposition matrices A ∈ ℝ^(d × r) and B ∈ ℝ^(r × k), where r ≪ min(d, k).

$$h = Wx + ΔWx = Wx + B(Ax)$$ During inference, B and A can be multiplied together and explicitly added to $W$, resulting in zero added latency.

3. QLoRA (Quantized LoRA)

import torch
import torch.nn as nn
import math

class LoRALinear(nn.Module):
    def __init__(self, linear_layer, r=8, alpha=16):
        super().__init__()
        self.linear = linear_layer
        self.r = r
        self.alpha = alpha
        self.scaling = alpha / r

        for param in self.linear.parameters():
            param.requires_grad = False

        in_features = self.linear.in_features
        out_features = self.linear.out_features

        self.lora_A = nn.Parameter(torch.zeros(in_features, r))
        self.lora_B = nn.Parameter(torch.zeros(r, out_features))

        nn.init.kaiming_uniform_(self.lora_A, a=math.sqrt(5))
        nn.init.zeros_(self.lora_B)

    def forward(self, x):
        base_out = self.linear(x)
        adapter_out = (x @ self.lora_A @ self.lora_B) * self.scaling
        return base_out + adapter_out

fc = nn.Linear(10, 5)
lora_fc = LoRALinear(fc, r=4)

from fastapi import FastAPI
from pydantic import BaseModel
from sentence_transformers import SentenceTransformer
import faiss

app = FastAPI()

corpus = ["FastAPI serves ML models asynchronously.", "Docker packages system dependencies.", "FAISS indices vectors in memory."]
corpus_embeddings = model.encode(corpus).astype('float32')
index = faiss.IndexFlatL2(384)
index.add(corpus_embeddings)

class QueryRequest(BaseModel):
    query: str

@app.post("/query")
async def retrieve_info(request: QueryRequest):
    q_embed = model.encode([request.query]).astype('float32')
    _, indices = index.search(q_embed, k=1)
    matched_text = corpus[indices[0][0]]
    return {"context": matched_text}

🛠️ WEEKLY DEBUGGING CHALLENGE

Debug: Missing Attention Scaling Factor

The self-attention output gradients vanish quickly during training, causing parameters to stop updating.

python
import torch
import math
Q = torch.randn(1, 8, 64) # Q shape
K = torch.randn(1, 8, 64) # K shape
# Compute raw dot-product attention weights
scores = torch.matmul(Q, K.transpose(-2, -1))
weights = torch.softmax(scores, dim=-1)
SOLUTION & EXPLANATION
The Bug: As the embedding dimension d_k grows large, the dot products grow large in magnitude, pushing the softmax function into regions with extremely small gradients. Dividing scores by sqrt(d_k) keeps logits scaled and prevents vanishing gradients.

The Fix: 
import torch
import math
Q = torch.randn(1, 8, 64)
K = torch.randn(1, 8, 64)
d_k = Q.size(-1)
# Scale scores by dividing by square root of key dimensions
scores = torch.matmul(Q, K.transpose(-2, -1)) / math.sqrt(d_k)
weights = torch.softmax(scores, dim=-1)
