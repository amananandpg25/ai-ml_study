1. Why Kubernetes for Production ML?

While Docker containerizes our application, Kubernetes orchestrates multiple containers across a cluster of servers, providing auto-scaling, load balancing, zero-downtime rolling updates, and self-healing. This is crucial for serving large ML workloads in production.

K8s Key Concepts:
• Pod: The smallest deployable unit, hosting one or more container instances.
• Deployment: Defines the desired state of pods (number of replicas, image versions).
• Service: Exposes pods to the network, load balancing traffic among them.
• kubectl: Command-line interface to interact with the K8s API.
graph TD Client["Client Request"] --> Service["K8s Service (LoadBalancer)"] Service --> Pod1["Pod 1 (ML API Container)"] Service --> Pod2["Pod 2 (ML API Container)"] Service --> Pod3["Pod 3 (ML API Container)"]

2. Declarative K8s Manifest YAML Example

Let's define a deployment.yaml to serve our Flask ML API container across 3 replicas:

yaml — deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: ml-model-deployment
  labels:
    app: ml-api
spec:
  replicas: 3
  selector:
    matchLabels:
      app: ml-api
  template:
    metadata:
      labels:
        app: ml-api
    spec:
      containers:
      - name: flask-ml-api
        image: username/flask-ml-model:v1
        ports:
        - containerPort: 5000
        resources:
          limits:
            cpu: "500m"
            memory: "512Mi"
          requests:
            cpu: "250m"
            memory: "256Mi"

3. Exposing the Deployment with a Service

Now, let's expose these pods via a LoadBalancer service using service.yaml:

yaml — service.yaml
apiVersion: v1
kind: Service
metadata:
  name: ml-model-service
spec:
  type: LoadBalancer
  selector:
    app: ml-api
  ports:
    - protocol: TCP
      port: 80
      targetPort: 5000

This GitHub Actions workflow file .github/workflows/deploy.yml automatically tests code before allowing merge:

yaml — deploy.yml
name: CI Pipeline
on: [push]
jobs:
  build-and-test:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v3
    - name: Set up Python
      uses: actions/setup-python@v4
      with:
        python-version: '3.10'
    - name: Install dependencies
      run: |
        pip install -r requirements.txt
    - name: Run Pytest

1. The Need for Experiment Tracking

During training, we adjust learning rates, model architectures, and data features. Without a centralized tracking tool, it's impossible to compare hundreds of training runs to select the best model. MLOps frameworks solve this.

2. Tracking Experiments with MLflow

Here is how to log parameters, metrics, and save our scikit-learn model artifact to MLflow registry:

python — train_mlflow.py
import mlflow
import mlflow.sklearn
from sklearn.ensemble import RandomForestClassifier
from sklearn.metrics import accuracy_score

mlflow.set_experiment("Customer_Churn_RF")

with mlflow.start_run():
    n_estimators = 100
    max_depth = 5
    
    # Log hyperparameters
    mlflow.log_param("n_estimators", n_estimators)
    mlflow.log_param("max_depth", max_depth)
    
    # Train
    model = RandomForestClassifier(n_estimators=n_estimators, max_depth=max_depth)
    model.fit(X_train, y_train)
    
    # Log metric
    acc = accuracy_score(y_val, model.predict(X_val))
    mlflow.log_metric("val_accuracy", acc)
    
    # Log model artifact
    mlflow.sklearn.log_model(model, "random_forest_model")
    print(f"Logged run with accuracy: {acc}")

🛠️ WEEKLY DEBUGGING CHALLENGE

Debug: Missing Docker CMD Instruction

A student built a multi-stage Dockerfile for their FastAPI app. However, the container exits immediately with code 0.

python
# Stage 2: Runtime
FROM python:3.10-slim
WORKDIR /app
COPY --from=builder /root/.local /root/.local
COPY . .
ENV PATH=/root/.local/bin:$PATH
EXPOSE 8000
SOLUTION & EXPLANATION
The Bug: A Docker container will exit immediately if there is no foreground blocking process keeping it alive. Forgetting to specify the CMD or ENTRYPOINT instruction leaves the container without a run target, so it shuts down immediately with exit code 0. You must specify CMD to start uvicorn.

The Fix: 
# Stage 2: Runtime
FROM python:3.10-slim
WORKDIR /app
COPY --from=builder /root/.local /root/.local
COPY . .
ENV PATH=/root/.local/bin:$PATH
EXPOSE 8000
CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]

SFT (Supervised Fine-Tuning) trains on instruction pairs. RLHF uses reinforcement learning on preference models for alignment.
LoRA Tuning?
Freezes base weights, injecting low-rank trainable weight matrices (A and B) to reduce fine-tuning memory by 99%.
Quantization?
Reduces model weight precision (e.g. FP32 to INT8/INT4) to decrease memory footprint and increase inference speed.
Catastrophic Forgetting?
When fine-tuning on a new task causes the model to lose its general capabilities or performance on previous tasks.
Parameter Efficiency?
Adapting foundation models with minimal weight updates, saving computing power and storage footprint.
