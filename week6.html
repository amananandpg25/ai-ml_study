1. The Full Linear Regression Story

Ghar ka rent predict karna: Size (sq ft) ≥ Area ≥ Floor se. Agar hum ek line draw karein jo saare observed rents ke "as close as possible" ho, woh line Linear Regression hai. "As close as possible" = minimize the sum of squared vertical distances between actual and predicted — yahi Ordinary Least Squares (OLS) hai.
$$ŷ = β₀ + β₁x₁ + β₂x₂ + ... + βₙxₙ = Xβ$$ Matrix form: X is (n×p) design matrix (with 1s column for intercept), β is (p×1) weight vector $$Loss (MSE) = (1/n) Σ(yᵢ − ŷᵢ)² = (1/n)||y − Xβ||²$$ We minimize this. Set derivative to zero → closed-form OLS solution: $$β* = (XᵀX)⁻¹ Xᵀy$$ The Normal Equation. Direct, exact, no iteration needed — but expensive for large n.
python — linear_regression_ols.py
import numpy as np
import pandas as pd
from sklearn.datasets import fetch_california_housing
from sklearn.model_selection import train_test_split
from sklearn.preprocessing import StandardScaler

# Load California Housing dataset (classic regression benchmark)
data = fetch_california_housing()
X_df = pd.DataFrame(data.data, columns=data.feature_names)
y    = data.target   # median house value in $100k

# Use just 2 features for visualisation clarity
X_raw = X_df[['MedInc', 'AveRooms']].values

X_train, X_test, y_train, y_test = train_test_split(
    X_raw, y, test_size=0.2, random_state=42)

# Scale features — critical before OLS with multiple features
sc = StandardScaler()
X_tr = sc.fit_transform(X_train)
X_te = sc.transform(X_test)

# ── OLS from scratch using Normal Equation ──────────────────
# Add bias column (column of 1s)
X_tr_b = np.column_stack([np.ones(len(X_tr)), X_tr])   # (n, 3)
X_te_b = np.column_stack([np.ones(len(X_te)), X_te])

# β* = (XᵀX)⁻¹ Xᵀy
beta = np.linalg.inv(X_tr_b.T @ X_tr_b) @ X_tr_b.T @ y_train

print("OLS Coefficients (scratch):")
print(f"  Intercept : {beta[0]:.4f}")
print(f"  MedInc    : {beta[1]:.4f}")
print(f"  AveRooms  : {beta[2]:.4f}")

# Predictions and MSE
y_pred_scratch = X_te_b @ beta
mse_scratch = np.mean((y_test - y_pred_scratch)**2)
print(f"
Test MSE (scratch OLS): {mse_scratch:.4f}")

# ── Verify against sklearn ───────────────────────────────────
from sklearn.linear_model import LinearRegression
lr = LinearRegression()
lr.fit(X_tr, y_train)
y_pred_sk = lr.predict(X_te)
mse_sk = np.mean((y_test - y_pred_sk)**2)
print(f"Test MSE (sklearn)     : {mse_sk:.4f}")  # should match!
print(f"sklearn coef_: {lr.coef_}")
print(f"sklearn intercept_: {lr.intercept_:.4f}")

2. The 5 Assumptions of Linear Regression (LINE + H)

📐 The LINE assumptions — violate them and your model lies: Linearity — the relationship between X and y IS linear.
Independence — observations are independent of each other (no time-series autocorrelation).
Normality — residuals (errors) are normally distributed.
Equal variance (Homoscedasticity) — residual variance is constant across all X values.
+H: No multicollinearity — features are not highly correlated with each other.

def regression_diagnostics(model, X_train, y_train, X_test, y_test):
    y_pred  = model.predict(X_test)
    residuals = y_test - y_pred

    fig, axes = plt.subplots(2, 2, figsize=(12, 8))
    fig.suptitle('Regression Diagnostic Plots', fontsize=14)

    # 1. Residuals vs Fitted — check Linearity + Homoscedasticity
    axes[0,0].scatter(y_pred, residuals, alpha=0.4, color='#6c8cff', s=20)
    axes[0,0].axhline(0, color='#e56b8c', linestyle='--')
    axes[0,0].set_xlabel('Fitted values'); axes[0,0].set_ylabel('Residuals')
    axes[0,0].set_title('Residuals vs Fitted')
    # WANT: random scatter around zero. Red flag: cone shape or curve.

    # 2. Q-Q Plot — check Normality of residuals
    stats.probplot(residuals, dist="norm", plot=axes[0,1])
    axes[0,1].set_title('Q-Q Plot (Normality Check)')
    # WANT: points on the diagonal line. Curves mean non-normal residuals.

    # 3. Scale-Location — check Homoscedasticity
    axes[1,0].scatter(y_pred, np.sqrt(np.abs(residuals)), alpha=0.4,
                      color='#4fd1a5', s=20)
    axes[1,0].set_xlabel('Fitted values'); axes[1,0].set_ylabel('√|Residuals|')
    axes[1,0].set_title('Scale-Location (Homoscedasticity)')
    # WANT: horizontal band. Upward funnel = heteroscedasticity (use WLS or log-transform y).

    # 4. Residual distribution histogram
    axes[1,1].hist(residuals, bins=40, color='#b47cfc', alpha=0.8, edgecolor='black')
    axes[1,1].set_xlabel('Residual value'); axes[1,1].set_title('Residual Distribution')
    # WANT: bell curve centred at zero.

    plt.tight_layout()
    plt.savefig('regression_diagnostics.png', dpi=150)
    plt.show()

    # Normality test
    stat, p = stats.shapiro(residuals[:5000])  # Shapiro-Wilk (max 5000)
    print(f"
Shapiro-Wilk normality test: p={p:.4f}")
    print("Residuals normal" if p > 0.05 else "Residuals NOT normal — consider transforming y")

python — multicollinearity_vif.py
import pandas as pd
import numpy as np
from sklearn.datasets import fetch_california_housing
from sklearn.linear_model import LinearRegression

# Variance Inflation Factor (VIF) — quantifies multicollinearity
# VIF = 1: no multicollinearity
# VIF 1-5: moderate (acceptable)
# VIF > 10: HIGH — problematic, remove this feature

def calc_vif(X_df):
    vif_data = {}
    for col in X_df.columns:
        # Regress col on all other columns → R²
        others = X_df.drop(columns=[col])
        model  = LinearRegression().fit(others, X_df[col])
        r2     = model.score(others, X_df[col])
        vif_data[col] = 1 / (1 - r2) if r2 < 1 else float('inf')
    return pd.Series(vif_data, name='VIF').sort_values(ascending=False)

data = fetch_california_housing()
X_df = pd.DataFrame(data.data, columns=data.feature_names)
vif = calc_vif(X_df)
print(vif)
print("
Features with VIF > 5 (potential multicollinearity):")
print(vif[vif > 5])

✅ Knowledge Check — Day 38 Quiz

QUESTION 1 OF 4
The Normal Equation β* = (XᵀX)⁻¹Xᵀy gives the exact OLS solution. Why might you still prefer gradient descent for large datasets?
A Gradient descent always finds a better solution than the Normal Equation
B Inverting (XᵀX) is O(p³) — with many features or samples it becomes prohibitively slow and memory-intensive
C The Normal Equation requires the data to be normally distributed
D Gradient descent can handle non-linear data but the Normal Equation cannot
✅ Correct! Matrix inversion of an (p×p) matrix is O(p³). With 10,000 features (common in NLP/genomics), that's 10¹² operations — computationally infeasible. Gradient descent updates are O(np) per step — much cheaper. sklearn uses SVD-based solver (not direct inversion) which is O(np²), faster and more numerically stable. For large n with small p, scipy.linalg.lstsq is even better.
❌ Normal Equation finds the globally optimal solution. The issue is computational cost: matrix inversion is O(p³). For 10k features, this is intractable. Gradient descent is iterative but scales to millions of features and samples. sklearn uses SVD instead of direct inversion — faster and more numerically stable than the pure Normal Equation.
QUESTION 2 OF 4
A residual plot shows a clear "fan" shape — residuals are small at low fitted values and large at high fitted values. Which assumption is violated?
A Linearity — the relationship isn't linear
B Homoscedasticity — residual variance increases with fitted values (heteroscedasticity)
C Normality — residuals aren't normally distributed
D Independence — observations are correlated
✅ Correct! A "fan" or "cone" shape in residuals vs fitted values is the classic signature of heteroscedasticity — the assumption of equal variance (homoscedasticity) is violated. Common with prices, incomes, counts. Fix: apply log(y) or sqrt(y) transformation to compress the scale. After transformation, recheck the plot — the fan should disappear.
❌ Fan shape = heteroscedasticity = violation of homoscedasticity. Linearity violation shows as a CURVE in the residuals. Normality violation shows as S-curves in the Q-Q plot or a skewed histogram. The fan pattern specifically means variance grows with the predicted value — log-transforming y usually fixes this.
QUESTION 3 OF 4
Feature A and Feature B have Pearson r = 0.97. You include both in a Linear Regression. What problem occurs?
A The model will overfit the training data
B Multicollinearity — coefficients become unstable, have large standard errors, and are uninterpretable
C The model will run significantly slower
D The predictions will be systematically biased
✅ Correct! Multicollinearity doesn't harm predictions much (predictions stay stable) but destroys coefficient interpretability. With r=0.97, XᵀX is nearly singular, making (XᵀX)⁻¹ numerically unstable — tiny changes in data cause huge swings in coefficients. The VIF for both features will be very high (>10). Fix: drop one of the two features, or use Ridge Regression which handles multicollinearity gracefully.
❌ Multicollinearity's main harm is to coefficient stability and interpretability, not predictions. With two highly correlated features, the model can't tell which one is responsible for the variation in y — so it assigns arbitrary, unstable weights to each. VIF quantifies this: VIF > 10 is the red flag. Ridge Regression is the standard fix because its L2 penalty regularizes the unstable coefficients.
QUESTION 4 OF 4
In linear regression, WHICH values must be normally distributed for valid statistical inference?
A The input features X
B The target variable y
C The residuals (y − ŷ) — not the features or target
D The predicted values ŷ
✅ Correct! This is one of the most common interview misconceptions. It's the RESIDUALS that must be approximately normal — not X, not y. Why? Because the t-tests on coefficients and confidence intervals assume the error term ε ~ N(0, σ²). If you're only using linear regression for prediction (not inference), this assumption matters less. But always check the Q-Q plot of residuals.
❌ Common misconception! Normality applies to RESIDUALS (y − ŷ), not to features or the target. You can have a skewed y and it's fine as long as the errors around the regression line are approximately normal. Check with: plt.hist(residuals) and scipy.stats.probplot(residuals). If residuals are non-normal, consider transforming y (log, sqrt) or switching to a GLM.

2. Bias-Variance Tradeoff — The Central Tension of ML

$$Expected Test Error = Bias² + Variance + Irreducible Noise$$ Three separate sources of error. Only bias and variance are controllable. $$Bias: error from wrong assumptions (e.g., fitting a line to a curve)$$ $$Variance: sensitivity to small fluctuations in training data$$

⚡ The Most Important Rule in ML Model Selection: Never select hyperparameters (like polynomial degree) based on training error — it always decreases with complexity. Always use cross-validation or a held-out validation set. The optimal degree is where VALIDATION error is minimized, not where training error is lowest. This rule applies to every hyperparameter in every model.
The bias-variance tradeoff is why regularization (Ridge, Lasso) exists. Both reduce variance at the cost of a small increase in bias — the net effect is lower test error. Deep learning models are extremely high-variance (millions of parameters) and use dropout, batch norm, weight decay, and data augmentation to control variance. The tradeoff never disappears — managing it is the core skill of ML engineering.

$$OLS Loss = Σ(yᵢ − ŷᵢ)²$$ $$Ridge Loss = Σ(yᵢ − ŷᵢ)² + α × Σβⱼ² [L2 penalty — squared coefficients]$$ $$Lasso Loss = Σ(yᵢ − ŷᵢ)² + α × Σ|βⱼ| [L1 penalty — absolute coefficients]$$ α (alpha) = regularisation strength. α=0 → pure OLS. α→∞ → all coefficients → 0.

2. WHY L1 Creates Zeros and L2 Doesn't — The Geometry

📐 The Geometry of Sparsity (Diamond vs. Circle):
• L1 Regularization (Lasso): Adds penalty $\lambda \sum |w_j|$. The constraint boundary forms a **diamond shape** (L1-ball) in weight space with sharp corners on the axes. The contours of the loss function expand until they intercept the constraint boundary. Mathematically, these contours hit the sharp corners on the axes first, forcing some weights to be exactly 0.
• L2 Regularization (Ridge): Adds penalty $\lambda \sum w_j^2$. The constraint boundary forms a **smooth circle/sphere** (L2-ball). The loss function contours intercept the circle at arbitrary non-zero tangent points. This shrinks all weights close to 0, but never sets them exactly to 0.
📐 The Geometry Explanation (Crucial for Interviews): Minimizing the regularised loss is geometrically equivalent to minimizing the OLS loss SUBJECT TO a constraint on the coefficients.

Ridge (L2): Constraint region is a CIRCLE (in 2D). The OLS solution contours touch the circle at curved points — almost never exactly at zero. Coefficients shrink toward zero but rarely reach it.

Lasso (L1): Constraint region is a DIAMOND (in 2D). The OLS solution contours are likely to touch the diamond at a CORNER — which in 2D is exactly at (0, β₂) or (β₁, 0). In high dimensions, corners are very common → MANY zeros → automatic feature selection.

QUESTION 2 OF 4
Ridge Regression has a closed-form solution: β = (XᵀX + αI)⁻¹Xᵀy. What does adding αI accomplish?
A It normalizes the features to unit variance
B It makes the matrix invertible even when XᵀX is singular or near-singular, and shrinks coefficients
C It adds a new feature (the bias term) to the design matrix
D It removes multicollinearity by orthogonalising the features
✅ Correct! Adding αI to XᵀX shifts all eigenvalues by α — ensuring none can be zero or near-zero. ❌ Adding αI to the diagonal of XᵀX increases all eigenvalues by α. This ensures the matrix is positive definite and invertible — even when XᵀX is singular (n<p, perfect multicollinearity). This is the fundamental mathematical advantage of Ridge over OLS. Geometrically, it also shrinks the estimated coefficients toward zero proportional to α.

✅ Correct! Ridge penalises β². import numpy as np
from sklearn.preprocessing import StandardScaler
from sklearn.linear_model import Ridge
from sklearn.model_selection import cross_val_score, KFold
from sklearn.pipeline import Pipeline
from sklearn.datasets import fetch_california_housing

data = fetch_california_housing()
X, y = data.data, data.target

# ❌ WRONG: Scaling BEFORE cross-validation = DATA LEAKAGE
# StandardScaler learns mean/std from ALL data including test folds!
sc_wrong = StandardScaler()
X_wrong  = sc_wrong.fit_transform(X)   # leaks test fold stats into training!
scores_wrong = cross_val_score(Ridge(), X_wrong, y, cv=5,
                               scoring='neg_mean_squared_error')
print(f"Leaky MSE (optimistic):  {-scores_wrong.mean():.4f}")

# ✅ CORRECT: Scaling INSIDE Pipeline — scaler fits on each training fold
pipe_correct = Pipeline([
    ('scaler', StandardScaler()),
    ('ridge',  Ridge(alpha=1.0))
])
scores_correct = cross_val_score(pipe_correct, X, y, cv=5,
                                 scoring='neg_mean_squared_error')
print(f"Correct MSE (honest):    {-scores_correct.mean():.4f}")
# The difference might look small here but on high-variance preprocessing
# steps (like imputation, encoding) leakage gives MASSIVELY optimistic estimates.

# === Nested Cross-Validation — truly unbiased evaluation ===
from sklearn.model_selection import cross_val_score, GridSearchCV
inner_cv = KFold(n_splits=5, shuffle=True, random_state=1)
outer_cv = KFold(n_splits=5, shuffle=True, random_state=2)

param_grid = {'ridge__alpha': [0.1, 1.0, 10.0]}
search = GridSearchCV(pipe_correct, param_grid, cv=inner_cv,
                      scoring='neg_mean_squared_error')
nested_scores = cross_val_score(search, X, y, cv=outer_cv,
                                scoring='neg_mean_squared_error')
print(f"
Nested CV MSE: {-nested_scores.mean():.4f} ± {nested_scores.std():.4f}")
# This is the gold standard: inner CV tunes alpha, outer CV evaluates performance.

If you scale ALL data before cross_val_score, you've let test fold statistics influence the scaler — artificially inflating performance estimates. The fix is always: put EVERYTHING in a Pipeline.

2. R² — The Most Misunderstood Metric

⚡ Why $R^2$ Can Be Negative:
The Coefficient of Determination formula is $R^2 = 1 - \frac{SS_{res}}{SS_{tot}} = 1 - \frac{\sum (y_i - \hat{y}_i)^2}{\sum (y_i - \bar{y})^2}$.
If your model predicts values that are *worse* than just predicting the horizontal average line (the mean $\bar{y}$) for every data point, then $SS_{res} > SS_{tot}$.
In this case, the ratio $\frac{SS_{res}}{SS_{tot}} > 1$, resulting in a negative value for $R^2$. A negative $R^2$ indicates that your model fails to capture even the simplest mean baseline of the target data.
$$R² = 1 − SS_res / SS_tot$$ SS_res = Σ(yᵢ − ŷᵢ)² (residual sum of squares — model error) SS_tot = Σ(yᵢ − ȳ)² (total sum of squares — variance in data) $$R² = 0: model no better than predicting the mean$$ $$R² < 0: model WORSE than predicting the mean! (always possible with wrong model)$$ $$R² = 1: perfect predictions$$

import numpy as np
from sklearn.linear_model import HuberRegressor

# Huber Loss — combines MSE (smooth) and MAE (outlier-robust)
# δ (delta) is the threshold:
# |error| ≤ δ → use MSE (quadratic, differentiable, fast gradient)
# |error| > δ → use MAE (linear, doesn't explode for huge errors)

def huber_loss(y_true, y_pred, delta=1.35):
    errors = y_true - y_pred
    is_small = np.abs(errors) <= delta
    huber = np.where(is_small,
                     0.5 * errors**2,
                     delta * (np.abs(errors) - 0.5 * delta))
    return np.mean(huber)

y_true = np.array([10, 20, 30, 40, 500])  # 500 is an outlier
y_pred = np.array([12, 19, 31, 42, 100])

mse_loss   = np.mean((y_true - y_pred)**2)
mae_loss   = np.mean(np.abs(y_true - y_pred))
huber_val  = huber_loss(y_true, y_pred, delta=10)

print(f"MSE loss   : {mse_loss:.1f}  ← dominated by the 500 outlier")
print(f"MAE loss   : {mae_loss:.1f}  ← balanced")
print(f"Huber loss : {huber_val:.1f}  ← in between")

# sklearn's HuberRegressor — use when you have outliers in y

QUESTION 1 OF 4
A model predicts 5 house prices. One prediction is wildly wrong (₹1 lakh predicted vs ₹50 lakh actual). Which metric is MOST affected?
A MAE — because it measures absolute errors
B MSE — because it squares the error, making the outlier error ₹49 lakh → ₹2401 lakh² in the sum
C R² — because R² captures total variance
D MAPE — because it's percentage-based
✅ Correct! MSE squares errors — a ₹49 lakh error contributes 49² = 2401 to the MSE, versus 49 for MAE. One huge outlier can dominate MSE completely. This is WHY MSE is used when large errors are especially catastrophic (you want the model to really avoid them). For house prices where big errors are equally bad as many small ones, MAE or Huber loss is often more appropriate.
❌ MSE is most sensitive to outliers because it squares errors. Error of 49 → contributes 49²=2401 to MSE sum, but only 49 to MAE sum. One outlier can make MSE 100× worse while MAE stays proportional. This is by design when you want to penalise large errors heavily — but it's a liability when your data has natural outliers that you don't want to over-penalise.
QUESTION 2 OF 4
A model has R²=0.82 with 3 features. You add 5 random noise features and R² increases to 0.84. What does Adjusted R² do?
A Adjusted R² DECREASES — it penalises adding features that don't genuinely improve the model
B Adjusted R² stays the same — it's unaffected by the number of features
C Adjusted R² also increases — confirming the new features are useful
D Adjusted R² is undefined when R² > 0.8
✅ Correct! R² can only increase (or stay constant) when you add features — even noise features contribute marginally to fitting. This makes R² useless for feature selection. Adjusted R² = 1 − (1−R²)(n−1)/(n−p−1). As p increases with the same n, the (n−1)/(n−p−1) fraction grows, increasing the penalty. If new features don't improve SS_res enough to offset this, Adjusted R² falls — correctly signalling the features aren't worth adding.
❌ R² NEVER decreases when you add features (even noise features accidentally explain a tiny bit more variance). Adjusted R² penalises the additional parameter count: as p increases, the formula's correction factor grows. If a feature doesn't improve fit enough to offset its "cost", Adjusted R² falls — correctly identifying the feature as not worth including. Use Adjusted R² for feature selection, not plain R².
QUESTION 3 OF 4
You're building a demand forecasting model for a supermarket chain. Managers want to understand model performance. Which metric is best to report?

QUESTION 4 OF 4
R² = −0.45 on a test set. What does this mean?
A There's a bug — R² cannot be negative
B The model explains 45% of the variance in the wrong direction
C The model performs WORSE than simply predicting the mean — something is fundamentally wrong
D The model has negative correlation with the target
✅ Correct! R² = 1 − SS_res/SS_tot. When SS_res > SS_tot, R² < 0. This happens when your model's predictions are FURTHER from the true values than simply always predicting the mean. 1. Support Vector Regression (SVR)

Socho ek pipe se guzarna. SVR ek "tube" banata hai data ke around — tube ki width ε (epsilon) hai. Jo points tube ke andar hain unhe model ignore karta hai (zero loss). Sirf jo points tube ke bahar hain unhe penalise karta hai. Yahi SVR ka magic hai — small errors ko bilkul tolerate karna, sirf large errors pe dhyan dena.
$$SVR minimizes: ||w||²/2 + C × Σ(ξᵢ + ξᵢ*)$$ Subject to: |yᵢ − ŷᵢ| ≤ ε + slack (ξ ≥ 0) ε = tube width (tolerance), C = regularisation (smaller C = more tolerance), kernel = non-linear transformation

import numpy as np
import matplotlib.pyplot as plt
from sklearn.svm import SVR
from sklearn.preprocessing import StandardScaler
from sklearn.pipeline import Pipeline
from sklearn.model_selection import GridSearchCV

np.random.seed(42)
# Highly non-linear data: y = sin(x) + 0.5*cos(3x) + noise
X = np.sort(6 * np.random.rand(150, 1) - 3, axis=0)
y = np.sin(X).ravel() + 0.5 * np.cos(3*X).ravel() + np.random.randn(150) * 0.15

# SVR with RBF kernel — requires scaling!
svr_pipe = Pipeline([
    ('scaler', StandardScaler()),
    ('svr', SVR(kernel='rbf'))
])

# Tune C, gamma, epsilon
param_grid = {
    'svr__C':       [0.1, 1, 10, 100],
    'svr__gamma':   ['scale', 'auto', 0.1, 1],
    'svr__epsilon': [0.01, 0.1, 0.5]
}
grid = GridSearchCV(svr_pipe, param_grid, cv=5, scoring='neg_mean_squared_error',
                    n_jobs=-1)
grid.fit(X, y)
print(f"Best SVR params: {grid.best_params_}")
print(f"Best CV MSE:     {-grid.best_score_:.4f}")

# Key SVR parameters:
# C (regularisation): larger C → fit training data more strictly (less tolerance)
# epsilon: width of the ε-tube — errors within ε get 0 penalty
# gamma (RBF): controls "reach" of each training point
#   gamma='scale' → 1/(n_features × X.var()) — good default
